# 部署：該上傳什麼檔案？

## 一、用 GitHub + Vercel（建議）

**不用手動上傳檔案**，用 Git 把專案推到 GitHub，再讓 Vercel 從 GitHub 建置與部署。

### 要推上去的（會上傳到 GitHub）

- **整份專案**底下的所有**原始碼與設定**，例如：
  - `src/`（全部）
  - `public/`
  - `docs/`
  - `index.html`
  - `package.json`、`package-lock.json`
  - `vite.config.js`、`tailwind.config.js`、`postcss.config.js`
  - `vercel.json`（若有）
  - `README.md`、`.gitignore`

### 不要上傳的（已在 .gitignore）

- **`.env`**（內有 Supabase 金鑰，絕對不要推上 GitHub）
- `node_modules/`（部署時會用 `npm install` 重裝）
- `dist/`（部署時會用 `npm run build` 重建）

### 步驟簡述

1. 在專案根目錄：`git add .` → `git commit -m "..."` → `git push`（推到你的 GitHub repo）。
2. 到 Vercel 用「Import Git Repository」選這個 repo，Framework 選 **Vite**。
3. 在 Vercel **Settings → Environment Variables** 新增：
   - `VITE_SUPABASE_URL` = 你的 Supabase Project URL
   - `VITE_SUPABASE_ANON_KEY` = 你的 Supabase anon key
4. 儲存後觸發 **Redeploy**。

這樣就是「上傳整份專案（不含 .env / node_modules / dist）到 GitHub」，再由 Vercel 自動建置與部署。

---

## 二、若改用手動上傳（例如純靜態空間）

1. 在本機專案根目錄執行：`npm run build`
2. 會產生 **`dist/`** 資料夾（建好的靜態檔）。
3. 把 **`dist/` 裡面的「全部內容」**上傳到主機的網站根目錄。

注意：手動上傳時，主機必須能設定環境變數（或你改用其他方式注入 `VITE_SUPABASE_URL`、`VITE_SUPABASE_ANON_KEY`），否則連不到 Supabase。

---

## 三、總結

| 情境 | 要上傳／推送的 |
|------|----------------|
| **推送到 GitHub** | 整份專案（含 `src/`、`public/`、`docs/`、`package.json`、各 config），**不要** `.env`、`node_modules`、`dist` |
| **Vercel 部署** | 不需手動上傳檔案，連 GitHub 後在 Vercel 設好環境變數即可 |
| **手動上傳靜態主機** | 先 `npm run build`，再上傳 **`dist/` 裡的全部檔案** |

**重要**：`.env` 不要上傳到任何公開或共用的地方；Supabase 的 URL 與 anon key 改在部署平台（如 Vercel）的「環境變數」裡設定。
